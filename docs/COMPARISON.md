# Implementation comparison report

Snapshot generated by [`bench/report.ts`](../bench/report.ts) against the shared
fixture [`bench/scenarios/comparison.json`](../bench/scenarios/comparison.json)
(8 scenarios, 12 probes), with the **Analysis** section (C) written by hand.

**Methodology.** All four implementations run the **same** fixture over the
**same** HTTP contract (baseline :8080, opinionated :8091, simple :8092, maxxed
:8093 — LLM builds in live mode). Each probe's recalled context is scored by an
**LLM judge** (Claude Opus 4.8): `correct` (pass/fail) + graded `score` 0–1, with
deterministic `must_include` / `abstain` as a secondary signal. Reproduce with
`bash scripts/run-comparison.sh`.

## C. Analysis

### Headline
The control behaves as designed: **baseline 33%** vs **92–100%** for the LLM
builds. Among the three real contenders the *aggregate* is nearly tied (92/92/100
over only 12 probes — within noise), so the signal is in the **per-category
failures** and the **cost/latency** profile, not the headline number.

### Why the baseline fails (and why that's the point)
v0 is rule-based lexical recall with no embeddings — the deliberate floor. It
fails wherever phrasing dodges its regexes:
- **Extraction misses** → no memory to recall. "I'm a backend engineer at Stripe"
  and "just started as a PM at Notion" don't match its `I work at X` pattern, so
  `fe-1`/`fe-2` return nothing. "I drive a Tesla" (`xs-1`) and the budget summary
  (`bud-1`) likewise — nothing was extracted.
- **Brittle verbs**: it knows `moved to` but not "call Berlin home" (`para-1`) or
  "**moving** to Lisbon" (`mh-1`).
- It "passes" `noise-1` only by luck — it dumped diet facts, and the judge was
  lenient because it didn't invent a *team*. Real abstention isn't there.
This is not a backlog item — it's the control that proves extraction + semantics
are what move the needle.

### Where the three LLM builds differ
Two real, repeatable gaps separate otherwise-tied builds:

- **`opinionated` misses the opinion arc (`arc-1`, 0.4).** Its contradiction model
  adds two-way links and narrates *binary* flips well (oranges→apples in its own
  fixture), but it doesn't *synthesize* a 3-step arc (love → annoyed → "fine for
  big apps, Python for scripts") into a single current stance. **Why:** linking ≠
  summarizing a trajectory. **Where to improve:** collapse a contradiction/opinion
  chain into a current-stance summary at recall time.
- **`simple` misses noise abstention (`noise-1`, 0.0).** Its recall is
  always-on-profile (stable facts first, like the baseline) with a fixed-weight
  hybrid and no real abstention gate, so an off-topic query still surfaces the
  profile. **Why:** no relevance floor on the "stable facts" tier. **Where to
  improve:** add a query-relevance/abstention threshold to `/recall`.
- **`maxxed` is the only clean sweep (12/12)** — extract→reconcile, hybrid+RRF+LLM
  rerank, entity-graph multi-hop, abstention gate all firing together.

### Cost / latency — the real tradeoff surface
Quality is close; the price differs sharply.

| | recall latency | ingest latency | note |
|---|---|---|---|
| simple | **~0.5 s** (fastest) | ~4.1 s | light recall (no LLM in the hot path) |
| opinionated | ~3.8 s | **~9.9 s** (slowest ingest) | per-fact parallel reconcile = many Opus calls/turn |
| maxxed | **~5.3 s** (slowest recall) | ~4.1 s | LLM rerank + compaction on every recall |

All three compact context well (~32–43 tokens, comfortably under budget). So:
- **opinionated** buys quality with an expensive *ingest* path (multiple Opus
  round-trips per fact) — fine under the 60 s `/turns` budget, but costly.
- **maxxed** buys its top score with an expensive *recall* path (LLM in the hot
  loop on every query) — the place a latency-sensitive deployment would feel it.
- **simple** is the best latency/quality balance, held back only by abstention.

### What/where to improve next (no fixes applied yet)
- **opinionated** → opinion-arc synthesis; cut ingest cost/latency (batch the
  per-fact reconcile, fewer Opus calls).
- **simple** → abstention/relevance gate on recall (its one real gap).
- **maxxed** → trim recall latency/cost (rerank+compaction in the hot path);
  graduate multi-hop from one-hop to n-hop.
- **baseline** → none planned; it's the control.
- **Cross-cutting:**
  1. **The benchmark itself is too small** — 12 probes can't separate 92% from
     100% with confidence. Next round: fold in a real dataset (LongMemEval /
     LoCoMo from [`docs/research/benchmarks`](research/benchmarks)) and add
     adversarial abstention, n-hop, and more opinion-arc cases.
  2. **Judge leniency** (baseline "passing" `noise-1`) argues for a stricter
     abstention rubric and/or 2-of-3 judge voting.
  3. All three independently hit the same provider quirk (`claude-opus-4-8`
     rejects `temperature`; `ANTHROPIC_BASE_URL` needs `/v1` handling) — worth a
     shared note so the eventual promoted build standardizes it.

### Verdict
On this fixture, **maxxed** leads on quality and **simple** leads on
latency/simplicity; **opinionated** sits between with a distinctive
contradiction-linking design but the weakest opinion-arc handling and the
costliest ingest. No promotion decision yet — the next step is a bigger benchmark
to see whether maxxed's edge and simple's speed hold up at scale.
